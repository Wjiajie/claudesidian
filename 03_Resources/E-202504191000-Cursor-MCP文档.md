了解如何在 Cursor 的 Composer 功能中添加和使用自定义 MCP 工具

[模型上下文协议 (MCP)](https://modelcontextprotocol.io/introduction) 是一个开放协议，允许您在 Cursor 中为 Agentic LLM 提供自定义工具。

MCP 工具可能不适用于所有模型。MCP 工具仅在 Composer 的 Agent 中可用。

Cursor 实现了 MCP 客户端，该客户端支持任意数量的 MCP 服务器。Cursor 的 MCP 客户端支持 `stdio` 和 `sse` 传输。## 向 Cursor 添加 MCP 服务器

要向 Cursor 添加 MCP 服务器，请转到 `Cursor 设置` > `功能` > `MCP` ，然后单击 `+ 添加新的 MCP 服务器` 按钮。

这将打开一个包含要填写的表单的模态框。在 `类型` 下选择传输方式，并填写服务器的昵称（ `名称` ），以及运行命令或服务器的 URL，具体取决于传输方式。

例如，以下是如何配置 [MCP 快速入门天气服务器](https://github.com/modelcontextprotocol/quickstart-resources/tree/main/weather-server-typescript) 的方法，假设它已经构建并放置在 ` ~/mcp-quickstart/weather-server-typescript/build/index.js` 。在这种情况下，整个命令字符串是 `node ~/mcp-quickstart/weather-server-typescript/build/index.js` 。

![Image 1](https://mintlify.s3.us-west-1.amazonaws.com/cursor/images/advanced/mcp-add-stdio.png)

以下是如何配置 [MCP 示例工具](https://github.com/modelcontextprotocol/python-sdk/tree/main/examples/servers/simple-tool) 的方法，假设它在本地端口 `8765` 上运行。

![Image 2](https://mintlify.s3.us-west-1.amazonaws.com/cursor/images/advanced/mcp-add-sse.png)

对于 SSE 服务器，URL 应该是 SSE 端点的 URL，例如 `http://example.com:8000/sse` 。

对于 stdio 服务器，该命令应该是可以从终端运行的有效 shell 命令。如果您需要设置环境变量，我们建议您编写一个小包装脚本来设置环境变量，然后运行服务器。

添加服务器后，它应出现在 MCP 服务器列表中。您可能需要手动按下 MCP 服务器右上角的刷新按钮，才能填充工具列表。以下是加载（修改后的）天气和示例服务器后的工具列表的样子。

![Image 3](https://mintlify.s3.us-west-1.amazonaws.com/cursor/images/advanced/mcp-connected.png)

### 管理 MCP 服务器

可以从 MCP 设置页面编辑或删除 MCP 服务器。

### 项目特定的 MCP 配置

您可以使用 `.cursor/mcp.json` 配置项目特定的 MCP 服务器。该文件遵循以下格式：

复制

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/username/Desktop",
        "/Users/username/Downloads"
      ]
    }
  }
}
```## 在 Agent 中使用 MCP 工具

Composer Agent 将 **自动** 使用 MCP 设置页面上 `可用工具` 下列出的任何 MCP 工具，如果它确定它们是相关的。要故意提示工具使用，只需告诉 Agent 使用该工具，可以通过名称或描述来引用它。

### 工具批准

默认情况下，当 Agent 想要使用 MCP 工具时，它将显示一条消息，要求您批准：

![Image 4](https://mintlify.s3.us-west-1.amazonaws.com/cursor/images/advanced/mcp-mars-request.png)

用户可以展开消息以查看工具调用参数。

#### Yolo 模式

您可以启用 Yolo 模式，以允许 Agent 自动运行 MCP 工具，而无需批准，类似于执行终端命令的方式。在此处阅读有关 Yolo 模式以及如何启用它的更多信息 [here](https://cursordocs.com/agent#yolo-mode) 。

### 工具响应

当使用工具时，Cursor 将在聊天中显示响应。此图像显示了来自示例工具的响应，以及工具调用参数和工具调用响应的展开视图。

![Image 5](https://mintlify.s3.us-west-1.amazonaws.com/cursor/images/advanced/mcp-mars-response.png)
