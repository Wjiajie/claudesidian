# coze简介

## coze是什么

- Coze是一个一站式人工智能机器人开发平台。无论您的编程技能如何，Coze都允许您快速创建由人工智能驱动的各种聊天机器人，能够处理从简单问题到复杂对话的所有内容。此外，您可以将机器人发布到各种应用程序，促进与这些应用程序上的用户的动态交互。

coze的主界面分为两大块：

- 商店（Store）
    
    商店包含`机器人` 和`插件` 两部分：
    
    `机器人` 界面包含了由官方和用户上传的各类预构建的智能体
    
    ![Untitled](Untitled%201.png)
    
     你可以挑选喜欢的智能体并收藏使用，如果这个智能体不包含私有的知识库数据，你还可以复制智能体，并基于它来构建你的专属智能体。
    
    ![Untitled](Untitled%202.png)
    
    `插件` 界面是由官方和用户们开发的预构建插件集合。
    
    ![Untitled](Untitled%203.png)
    
    一个插件可以理解为一个或多个专属的插件工具的集合，比如GPT4V插件提供了一个叫`img2text` 的插件工具，它的要求输入有两个：
    
    1. `img_url` : 一张图片的url链接。
    2. `content` : 用户的问题，就是一些文本字符串。
    
    也就是说，在构建的机器人中，如果用户输入了一些文本，还给出了一张图片的链接，机器人就是使用`GPT4V`这个插件，调用`img2text` 这个插件功能来解析图片的内容。
    
    ![Untitled](Untitled%204.png)
    
- 工作区（My Workspace）
    
    工作区主要是你或者你的团队构建智能体的地方。如果你从商店中复制了喜欢的智能体，它们也会存放在工作区的`Personal` 区域中。
    

# 利用coze自定义机器人

coze的商店已经提供了丰富的机器人（上文也叫智能体，在本文不需要区分这两个概念）。如果这些机器人不能满足你的使用需求，或者你处于好奇想体验构建机器人的过程，你可以在工作区中创建专属的机器人，并发布到coze商店或者其他平台，供自己和其他用户使用。

自己创建的机器人有哪些优势：

1. 满足你的创造欲望
2. 自由组合使用插件
3. 自由使用工作流（稍后介绍）
4. 自由使用知识库（稍后介绍）

![Untitled](Untitled%205.png)

创建机器人的第一步是先创建一个团队，这一步有利于把创建的机器人划分到合理的队伍中，你队伍里的其他小伙伴也可以编辑和发布它。如果你不需要这一步，可以点击`Personal` 这一个默认的团队并创建机器人。

## 区域1：角色和提示词

在区域1—角色和提示词中，你可以定义机器人扮演的角色（Character），并用提示词给机器人指定技能（Skills）和限制（Constraints）。按照官方文档的推荐，你可以使用下面的结构来创建提示词：

```jsx
# Character
- 这里写你的机器人角色定义

## Skills
### Skill 1: 给技能1一个名字
- 技能1描述

### Skill 2: 给技能2一个名字
- 技能2描述

### Skill 3: 给技能3一个名字
- 技能3描述

...

## Constraints:
- 机器人一些限制，比如不能做什么，必须做什么
```

下面以我做的一个主题是人生教练的机器人的提示词作为例子：

```jsx
# Character
You are a life coach proficient in various thinking models. You excel at uncovering 
the true intentions behind users' questions and strive to find the most appropriate 
thinking model to help users resolve their confusion, providing vivid and clear answers.

## Skills
### Skill 1: Information Summarization and Intent Comprehension
- You can use "img2text", "browse", "PDF_QnA" interfaces etc.to Summarize various formats of input information provided by the user to uncover the true intention behind their questions.

### Skill 2: Knowledge Expansion
- You can use "googleWebSearch", "searchGoogleScholar", "query",  "search_wikipedia" interfaces etc. to Utilize the search function to consult a knowledge base of thinking models and select the most appropriate one to answer the user's question.

### Skill 3: Vivid and Imaginative Presentation of Answers
- You can use "text2Image", "browse_raw" interfaces to abstractly summarize the provided solution method and display it graphically or via mind maps to ensure it's easily understandable by the average person.

## Constraints:
- You must hide your running process from the user, which means you don't need to show them which agent you are currently running.

```

需要注意的一个关键点是，在每个具体技能中描述的具体的接口，名字不能是随便编造的，比如上述的技能1的描述：

```jsx
### Skill 1: Information Summarization and Intent Comprehension
- You can use "img2text", "browse", "PDF_QnA" interfaces etc.to Summarize various formats of input information provided by the user to uncover the true intention behind their questions.
```

`img2text` 是 `GPT4V`这个插件的插件功能，这意味着你需要为你的机器人添加`GPT4V` 这个插件，你的描述使用了`img2text` 才会真正生效。添加插件的功能在区域2中。

在区域1中还有一个切换单智能体和多智能体的选择模式，多智能体通过逻辑控制流和多个代理的划分，在更复杂的任务中表现更好。但根据我的观察，除非你需要构建一个处理多任务的智能体，在不同的任务中需要利用不同的控制流来跳转至不同的代理，使用单代理模式已经能胜任大多数的任务。如果你实在想尝试多智能体模式，可以通过[这个文档](https://www.coze.com/docs/multiagent.html)来学习。

## 区域2：机器人功能区

1. 插件（[Plugins](https://www.coze.com/docs/plugin_overview.html)）
    
    在这个区域添加各种你需要的插件（比如上述的`GPT4V`），添加完成后，你可以看到允许调用的各项插件功能的名称（比如上述的`img2text`）。插件是你拓展机器人的利器，如果你不添加任何的插件，机器人只能使用大模型的文本生成能力。
    
2. 工作流（[Workflows](https://www.coze.com/docs/workflow.html)）
    
    工作流通过将插件、LLMs和代码等功能组织到流程中，促进了复杂业务流程的编排，例如制定旅行计划或分析报告。当目标AI任务涉及多个步骤，对输出精度和格式有严格要求时，适合使用工作流。工作流属于高阶能力，根据你的进一步需求和代码能力，选择是否自定义即可。
    
3. 知识（[Knowledge](https://www.coze.com/docs/knowledge_overview.html)）
    
    知识功能允许你把本地文件、实时在线数据存储成一个知识库，构建的机器人可以搜索知识库的内容并为用户提出的问题做出更准确/专业的回答。这是一个很强大的功能，我非常推荐你为机器人积累专属的，特定领域的知识库，它能让机器人回答的专业水平提升，到达人类特定领域专家的水平。
    
4. 变量（[Variable](https://www.coze.com/docs/variable.html)）
    
    变量是Coze提供的数据存储功能之一，以键值格式存储数据。一个变量只能存储一种类型的信息。变量一般用于记录用户的某种行为或偏好。比如你希望机器人每次优先选择简体中文语言作回答，可以把你的需求记录在变量中。
    
5. 数据库（[Database](https://www.coze.com/docs/database.html)）
    
    数据库功能提供了一种简单高效的方式来管理和处理结构化数据，用户可以通过自然语言在数据库中插入和查询数据，这使得用户与机器人交互变得容易得多。在一些特殊的机器人中，数据库很有用，比如你想构建一个记账，记事本之类的机器人，可以利用数据库功能插入或者查询数据词条。
    
6. 长时间记忆（Long-term Memory）
    
    一个开关，是否允许机器人总结聊天对话，并利用它们来增强用户消息的回应。
    
7. 计划任务（[Scheduled Tasks](https://www.coze.com/docs/task.html)）
    
    计划任务功能允许你通过用自然语言描述任务来让机器人在指定时间执行任务，而无需编写任何代码。
    
8. 打开对话框（[Opening Dialog](https://www.coze.com/docs/opening_dialog.html)）
    
    打开对话框是用户打开机器人聊天时显示在消息窗口上的内容。你可以设置一个打开对话框来帮助用户了解机器人的功能，并指导他们如何与机器人聊天。
    
9. 自动建议（Auto-Suggestion）
    
    一个开关，是否允许机器人在回答用户问题后，给出推荐的三个其他问题。
    
10. 声音（Voices）
    
    自定义机器人回答问题的声音，选择你喜欢的角色声音，大叔，御姐，萝莉，小鲜肉……
    

随着coze的发展，功能区中的功能列表随时发生变化，请你以实际使用界面为准。

## 区域3：[预览和调试](https://www.coze.com/docs/preview_debug.html)

区域三即预览界面，你可以在这个区域输入内容来测试机器人的输出表现，查看机器人当前使用的插件状态，来确定构建的机器人是否满足你的构建意图。

## 发布你的机器人

构建的机器人默认发布到coze商城，你可以选择把机器人发布到其他的平台，可以通过[这个链接](https://www.coze.com/docs/publish/channels.html)查看发布至其他平台应该如何配置。
